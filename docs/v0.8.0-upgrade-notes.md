# Orchestra v0.8.0 — Upgrade Notes & Drop-in Guide

This bundle upgrades `lucky_orchestra-plugin` from v0.7.0 to v0.8.0. It changes **what makes
the run stop and what it knows before it starts** — without touching any of the 8 verified
mechanisms or the v0.7.0 scoping/durability pillars.

## What changed, in one paragraph

A blank-page run now **builds knowledge first** (Stage 0.5 Landscape, with ChatGPT Deep
Research pointed at the survey, not just the plan), so the CHALLENGE gate is grounded instead
of guessing. Research now has a **checkable coverage standard** (source-type, recency,
triangulation) and a **missing-coverage review lens** (Persona D). A process model's own
numbers are **checked deterministically** before any review is spent (`physics_check.py`:
mass/energy closure, 2nd-law/no temp-cross always; ΔT_min and other design criteria
dynamically, as they are *formed* over rounds and logged in `decisions[]`). And the decide
gate now stops on **convergence**, not a counter: a passing score must be **confirmed** by an
independent blind pass, a **papered-over disagreement blocks PASS**, and at the round limit
the gate **extends if converging / escalates to a human if stalled** — it never auto-passes
to end the loop.

## Migration (backward compatible)

- A v0.7.0 `meta.json` still evaluates — missing v0.8.0 fields are treated as empty/false.
- To activate the new guards, add to `meta.json`: `research_standards.recency_floor`,
  `topics[].required_sources`, `disagreements[]`, and per round `confirm_score` +
  `major_findings`. (A round without `major_findings` cannot PASS — by design — until you
  record it.)
- `pass_threshold` default is now 8 (was 7). Set it explicitly to 7 for internal-grade work.
- The physics check runs **only** when a `streams.json` exists in the run dir, so non-
  process-model deliverables (e.g. investment theses) are unaffected.

## `streams.json` contract (for process-model runs)

The Synthesizer emits this beside the draft. Minimal shape:

```json
{
  "tol": {"mass_pct": 0.1, "energy_pct": 2.0},
  "streams": [
    {"id":"S1","role":"in","mdot_kg_s":1.0,"T_K":300,"P_bar":2.0,"h_kJ_kg":100.0,"components":{"H2":1.0}}
  ],
  "units": [{"id":"K1","in":["S1"],"out":["S2"],"Q_kW":0.0,"W_kW":120.0}],
  "heat_exchangers": [{"id":"HX1","hot_in":"S2","hot_out":"S3","cold_in":"C1","cold_out":"C2"}]
}
```

`role` ∈ in|out|internal. Unit energy sign convention: `Σ(mdot·h)_in + Q_kW + W_kW = Σ(mdot·h)_out`
(Q,W are heat/work ADDED to the stream). To enforce a design criterion, add a `check` to a
decision, e.g. `{"type":"dt_min","hx":"HX1","min_K":3}`.

## Honest limits (carried forward, see SKILL.md)

Physics closure proves **consistency, not correctness** — fabricated-but-consistent numbers
still pass Layer 1; the Reviewer + web audit catch wrongness. The gate trusts the numbers you
transcribe (traces make lapses auditable; the confirmation pass makes a single lenient score
insufficient). Manual ChatGPT load scales with rounds — a thorough paper run is a focused
work session, not one click.
